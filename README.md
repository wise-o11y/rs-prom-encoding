# rs-prom-encoder

Prometheus-compatible chunk encoder for XOR float and native histogram time series.

Produces chunk binaries that are **byte-identical** to Prometheus
[`tsdb/chunkenc`](https://pkg.go.dev/github.com/prometheus/prometheus/tsdb/chunkenc),
so chunks encoded in Rust can be read by any Go-based Prometheus ecosystem tool
(Prometheus, Thanos, Mimir, Cortex, etc.).

## Supported Encodings

| Encoding | Value | Description |
|----------|-------|-------------|
| `EncXOR` | 1 | Gorilla-based XOR float chunk encoding |
| `EncHistogram` | 2 | Native integer histogram chunk encoding |
| `EncFloatHistogram` | 3 | Native float histogram chunk encoding |

## Usage

### XOR Float Chunks

```rust
use rs_prom_encoder::XORChunk;

let mut chunk = XORChunk::new();
chunk.append(1000, 1.5);
chunk.append(2000, 2.5);
chunk.append(3000, 3.5);

let bytes = chunk.encode();
```

### Native Integer Histogram Chunks

```rust
use rs_prom_encoder::{HistogramChunk, Histogram, Span, CounterResetHint};

let mut chunk = HistogramChunk::new();
chunk.append(1000, &Histogram {
    count: 10,
    zero_count: 2,
    sum: 18.4,
    schema: 3,
    zero_threshold: 2.938735877055719e-39, // 2^-128 (Prometheus default)
    positive_spans: vec![Span { offset: 0, length: 2 }],
    negative_spans: vec![],
    positive_buckets: vec![4, 4],
    negative_buckets: vec![],
    custom_values: vec![],
    counter_reset_hint: CounterResetHint::UnknownCounterReset,
});

let bytes = chunk.encode();
```

### Native Float Histogram Chunks

```rust
use rs_prom_encoder::{FloatHistogramChunk, FloatHistogram, Span, CounterResetHint};

let mut chunk = FloatHistogramChunk::new();
chunk.append(1000, &FloatHistogram {
    count: 10.0,
    zero_count: 2.0,
    sum: 18.4,
    schema: 3,
    zero_threshold: 2.938735877055719e-39,
    positive_spans: vec![Span { offset: 0, length: 2 }],
    negative_spans: vec![],
    positive_buckets: vec![4.0, 4.0],
    negative_buckets: vec![],
    custom_values: vec![],
    counter_reset_hint: CounterResetHint::UnknownCounterReset,
});

let bytes = chunk.encode();
```

## Architecture

XOR encoding is backed by [`rusty-chunkenc`](https://crates.io/crates/rusty-chunkenc).
Histogram encoding is implemented from scratch to match Prometheus byte-for-byte.

```
rs-prom-encoder/
├── src/
│   ├── lib.rs                    # Public API
│   ├── encoding.rs               # Encoding enum, constants
│   ├── xor_chunk.rs              # XOR chunk (via rusty-chunkenc)
│   ├── bstream.rs                # Bit-level stream I/O
│   ├── varbit.rs                 # Variable-bit + LEB128 encoding
│   ├── xor.rs                    # XOR float encoding (for histograms)
│   ├── histogram_types.rs        # Histogram/Span/CounterResetHint types
│   ├── histogram_meta.rs         # Layout: spans, zero threshold, custom bounds
│   ├── histogram_chunk.rs        # Integer histogram chunk encoder
│   └── float_histogram_chunk.rs  # Float histogram chunk encoder
├── tests/fixtures/               # Golden binary fixtures from Go
├── go-golden/                    # Go program to generate golden fixtures
└── benches/                      # Criterion benchmarks
```

## Compatibility

Byte-level compatibility is verified against golden fixtures generated by
Prometheus's own Go `chunkenc` library. The test harness encodes identical
samples in both Rust and Go and asserts the output bytes match exactly.
